# Weverse 유저스토리 맵 (User Story Map)

> Phase 2 산출물 | 분석일: 2026-03-11
> 형식: Epic → User Story (As a... I want to... So that...)
> 각 스토리에 Supabase + Expo 구현 메모 포함
> 총 유저스토리: **62개**

## Epic 1: 온보딩 & 계정 관리

### US-01. 회원가입
| 항목 | 내용 |
|------|------|
| **페르소나** | P1 일반팬 |
| **스토리** | 팬으로서 소셜 계정(Google/Apple/Twitter)으로 간편하게 가입하고 싶다. 그래야 빠르게 좋아하는 아티스트 커뮤니티에 참여할 수 있다. |
| **인수 조건** | 소셜 로그인 3종 지원, 이메일 인증, 닉네임 설정, 이용약관 동의 |
| **Supabase+Expo 메모** | Supabase Auth (Google/Apple OAuth), `expo-auth-session`으로 네이티브 OAuth 플로우. `profiles` 테이블에 nickname, avatar_url, language 저장 |

### US-02. 프로필 설정
| 항목 | 내용 |
|------|------|
| **페르소나** | P1 일반팬 |
| **스토리** | 팬으로서 프로필 사진과 닉네임, 소개를 설정하고 싶다. 그래야 커뮤니티에서 나만의 정체성을 표현할 수 있다. |
| **인수 조건** | 닉네임 중복 검사, 프로필 이미지 업로드, 자기소개 텍스트, 언어 설정 |
| **Supabase+Expo 메모** | Supabase Storage (avatars 버킷), `expo-image-picker`로 이미지 선택, RLS로 본인 프로필만 수정 가능 |

### US-03. 커뮤니티 탐색 & 가입
| 항목 | 내용 |
|------|------|

| **인수 조건** | 아티스트 검색 (이름/키워드), 커뮤니티 프리뷰, 1-tap Join, 멀티 커뮤니티 가입 |

| **인수 조건** | 선호 언어 설정, 게시글/댓글 자동번역 토글, 원문/번역 전환 |
| **Supabase+Expo 메모** | Edge Function에서 DeepL/Google Translate API 호출, 번역 결과를 `post_translations` 테이블에 캐싱 (locale, translated_text) |

## Epic 3: 아티스트 콘텐츠

### US-12. 아티스트 게시글 열람

| **인수 조건** | Artist 탭 분리, 게시 시간순, 이미지/영상/텍스트, 자동번역 |
| **Supabase+Expo 메모** | `posts` where `author_role='artist'` AND `community_id=X`, RLS로 artist role만 artist 탭에 게시 가능 |

### US-13. 아티스트 Moments (스토리)
| 항목 | 내용 |
|------|------|

| **인수 조건** | 24시간 후 만료 [추정], 이미지/짧은 영상, 조회 수 표시 |
| **Supabase+Expo 메모** | `moments` 테이블 (expires_at), pg_cron으로 만료 처리, 스토리 UI는 react-native-story-view 등 |

| **인수 조건** | 댓글 작성, 아티스트 답글 하이라이트 표시, 자동번역 |

| **인수 조건** | 텍스트/이미지/영상 첨부, 커뮤니티 선택, 게시 전 미리보기 |
| **Supabase+Expo 메모** | `posts` INSERT with role check (RLS), 게시 시 구독자 전원에게 Push 발송 (Edge Function fan-out) |

| **인수 조건** | 팬 피드 브라우징, 댓글 작성 (아티스트 마크 자동 표시), 팬에게 알림 발송 |

## Epic 4: LIVE 스트리밍

### US-17. LIVE 시청
| 항목 | 내용 |
|------|------|
| **페르소나** | P1 일반팬 |
| **스토리** | 팬으로서 아티스트의 실시간 라이브 방송을 보고 싶다. 그래야 아티스트와 실시간으로 같은 시간을 공유할 수 있다. |
| **인수 조건** | 실시간 영상 재생, 시청자 수 표시, 하트/이모지 리액션 |
| **Supabase+Expo 메모** | Mux 또는 AWS IVS 스트리밍, `expo-av` 플레이어, 시청자 수는 Supabase Realtime Presence |

### US-18. LIVE 실시간 채팅
| 항목 | 내용 |
|------|------|
| **페르소나** | P1 일반팬 |

| **인수 조건** | 실시간 메시지 전송/수신, 자동번역, 하트 애니메이션 |
| **Supabase+Expo 메모** | Supabase Realtime Channel (live_{stream_id}), 메시지 rate limiting (Edge Function), 채팅 로그는 `live_chats` 테이블 |

### US-19. LIVE 하트 보내기
| 항목 | 내용 |
|------|------|
| **페르소나** | P1 일반팬 |
| **스토리** | 팬으로서 LIVE 중 하트를 연속으로 보내고 싶다. 그래야 아티스트에게 사랑과 응원을 표현할 수 있다. |
| **인수 조건** | 탭으로 하트 전송, 하트 떠오르는 애니메이션, 하트 수 집계 |
| **Supabase+Expo 메모** | 클라이언트 로컬에서 애니메이션 + 일정 주기로 배치 전송 (debounce), Realtime broadcast로 다른 시청자에게 동기화 |

### US-20. LIVE 시작 (아티스트용)
| 항목 | 내용 |
|------|------|
| **페르소나** | P3 아티스트 |
| **스토리** | 아티스트로서 1-tap으로 라이브 방송을 시작하고 싶다. 그래야 부담 없이 언제든 팬과 실시간 소통할 수 있다. |
| **인수 조건** | 라이브 시작/종료, 제목 입력, 카메라 전환, 전체 구독자 Push 알림 |
| **Supabase+Expo 메모** | Mux/IVS stream key 발급 (Edge Function), `live_streams` 테이블 (artist_id, status, started_at), 시작 시 fan-out Push |

### US-21. LIVE VOD 다시보기

| **인수 조건** | LIVE 종료 후 자동 VOD 전환, 채팅 리플레이, 탐색 재생 |
| **Supabase+Expo 메모** | Mux/IVS 녹화 기능, VOD URL을 `live_streams.vod_url`에 저장, 채팅은 타임스탬프 기반 리플레이 |

## Epic 5: Weverse DM

### US-22. DM 아티스트 구독

### US-23. DM 메시지 수신
| 항목 | 내용 |
|------|------|

| **인수 조건** | 텍스트/사진/영상/음성 메시지 수신, Push 알림, 닉네임으로 호칭 |
