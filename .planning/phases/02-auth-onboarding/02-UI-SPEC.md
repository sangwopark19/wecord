## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, dot indicator spacing |
| sm | 8px | Icon-to-text gap, button internal vertical padding |
| md | 16px | Screen horizontal padding, card internal padding |
| lg | 24px | Section vertical spacing between onboarding steps |
| xl | 32px | Logo-to-button gap on login screen |
| 2xl | 48px | Top margin above logo on login screen |
| 3xl | 64px | Bottom safe-area clearance |

Exceptions:
- Touch targets for OAuth buttons: minimum 44px height (iOS HIG requirement)
- Creator curation grid cards: 12px gap between cards (3px offset from 8-point grid — nearest valid is 12px)
- Dot indicator dots: 8px diameter, 4px gap between dots

Source: WEVERSE-UI-UX-GUIDE §1.3, iOS Human Interface Guidelines minimum touch target

## Color

All values pre-populated from `apps/mobile/tailwind.config.js` and WEVERSE-UI-UX-GUIDE §1.1:

| Role | Hex | Tailwind Token | Usage |
|------|-----|----------------|-------|
| Dominant (60%) | `#000000` | `bg-background` / `text-foreground` | All screen backgrounds, modal backdrops |
| Secondary (30%) | `#1A1A1A` | `bg-card` | Creator curation cards, ToS scroll container, language picker row background |
| Input surface | `#2A2A2A` | `bg-input` / `border-border` | Date of birth input field, divider lines |
| Muted text | `#999999` | `text-muted-foreground` | Secondary copy under headings, placeholder text, skip link |
| Subtle text | `#666666` | `text-subtle` | Inactive dot indicator, legal disclaimer |
| Accent (10%) | `#00E5C3` | `text-teal` / `bg-teal` | See reserved list below |
| Accent dark | `#00B89A` | `bg-teal-dark` | OAuth button pressed state |
| Destructive | `#FF4444` | `text-destructive` | Account deletion text (not in Phase 2 — defined for completeness) |
| Badge | `#FF3B30` | `bg-badge` | Not used in Phase 2 |
| White | `#FFFFFF` | `text-foreground` | Primary text on all dark surfaces |

Accent (`#00E5C3`) reserved for ONLY these elements:
1. "계속하기" / "다음" primary CTA button background
2. Selected state ring on creator curation cards (2px border)
3. Active dot in the onboarding progress dot indicator
4. Checkmark icon inside selected curation cards
5. "Google로 시작하기" button text (Google button uses white background with teal text — NOT filled teal)

The accent is NOT used for: general links, inactive states, background fills outside the items above.

Source: `apps/mobile/tailwind.config.js`, CONTEXT.md §Implementation Decisions, WEVERSE-UI-UX-GUIDE §1.1

## Interaction Contracts

### Screen Transition

| Transition | Animation |
|-----------|-----------|
| Login → ToS | Slide left (standard push), 300ms ease-in-out |
| ToS → DoB | Slide left, 300ms ease-in-out |
| DoB → Language | Slide left, 300ms ease-in-out |
| Language → Curation | Slide left, 300ms ease-in-out |
| Curation → Home | Fade, 400ms ease-in-out (major context change) |
| Back navigation | Slide right (standard pop) |

Source: CONTEXT.md "온보딩 애니메이션/전환 효과" — in Claude's discretion; standard push/pop with fade at completion is consistent with Weverse UX

### Skip Behavior (Curation Screen)

- Skip link: top-right corner, "건너뛰기" text 14px weight 400 `text-muted-foreground`
- Tap skip: navigate to home immediately, no confirmation dialog
- 0 selections + "다음": same as skip (allowed per CONTEXT.md "0개부터 원하는 만큼")

### ToS Acceptance

- Checkbox: 20px square, `#2A2A2A` background unselected, `#00E5C3` fill + white checkmark selected
- "다음" button is disabled until checkbox is checked
- ToS text scrollable in container, 240px max height before scroll activates

### Date of Birth

- Native date picker (iOS DatePickerIOS / Android date picker via `@react-native-community/datetimepicker`)
- Display format: YYYY-MM-DD in the input field
- Age gate: under 14 shows blocking message (see Copywriting Contract)

## Error & Loading States

| State | Visual Treatment |
|-------|-----------------|
| OAuth loading | OAuth button shows ActivityIndicator (white, 20px) replacing button text; button disabled |
| Curation loading | Skeleton cards (shimmer effect, `#2A2A2A` base → `#3A3A3A` shimmer) in 2-column grid |
| Network error | Bottom toast: `#1A1A1A` background, white text 14px, `#FF4444` left border 3px, auto-dismiss 4s |
| Auth error | Same toast pattern as network error |
| Age gate (under 14) | Inline text below date input, 12px label `#FF4444`, "다음" button remains disabled |
| Session restore (app open) | Full-screen `bg-background` with centered Wecord logo, no spinner (fast path) |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| OAuth button touch targets | Minimum 44px height (both buttons set to 52px) |
| Color contrast (teal on black) | `#00E5C3` on `#000000` — 6.9:1 ratio, exceeds WCAG AA (4.5:1) |
| Color contrast (white on card) | `#FFFFFF` on `#1A1A1A` — 14.7:1 ratio, exceeds WCAG AAA |
| Disabled button | Must not use color alone — "다음" shows `#666666` text + `#2A2A2A` bg (distinct from enabled) |
| Screen reader labels | All OAuth buttons need `accessibilityLabel` matching visible text |
| Date picker | Use platform native date picker — inherits OS accessibility |
