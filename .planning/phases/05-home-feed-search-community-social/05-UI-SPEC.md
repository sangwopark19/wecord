## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, chip internal gap |
| sm | 8px | Between cards in grid, gap between action bar items |
| md | 16px | Screen horizontal padding, section body padding |
| lg | 24px | Section header top margin, profile stat block spacing |
| xl | 32px | Major vertical gap between feed sections |
| 2xl | 48px | Page-level top padding under safe area |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets (Pressable, icon buttons): minimum 44px × 44px enforced via explicit `style` — matches existing `NotificationBellBadge` and `PostCard` action bar pattern
- Search bar height: 44px (matches `(community)/search.tsx` pattern)
- Primary CTA button height: 52px (matches `PrimaryCTAButton` component)
- Promotion banner carousel: aspect ratio 16:9 (width: 100%, calculated height)
- Community card cover image: aspect ratio 1:1 (matches `CommunityCard` pattern)

> Source: `apps/mobile/components/post/PostCard.tsx`, `apps/mobile/components/community/CommunityCard.tsx`, `apps/mobile/components/PrimaryCTAButton.tsx`

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px (`text-label`) | 400 regular | 1.4 |
| Body | 14px (`text-body`) | 400 regular | 1.5 |
| Heading | 16px (`text-heading`) | 600 semibold | 1.3 |
| Display | 20px (`text-display`) | 600 semibold | 1.2 |

Rules:
- Only two weights are used: 400 (`font-regular`) and 600 (`font-semibold`). No other weights.
- Community nickname on profile header: `text-display font-semibold` (20px / 600)
- Post body text: `text-body font-regular` (14px / 400)
- Community name in 2-column grid card: `text-heading font-semibold` (16px / 600)
- Stat counts (posts / followers / following) on profile: `text-heading font-semibold` (16px / 600)
- Stat labels below counts: `text-label` (12px / 400) `text-muted-foreground`
- Section header (e.g. "인기 크리에이터를 만나보세요!"): `text-heading font-semibold` (16px / 600)
- Notification badge counter: 10px / 600 (inline style — matches existing `NotificationBellBadge`)

> Source: `apps/mobile/tailwind.config.js` (fontSize declarations), confirmed by PostCard.tsx and CommunityCard.tsx usage

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (`bg-background`) | Screen background, SafeAreaView fill |
| Secondary (30%) | `#1A1A1A` (`bg-card`) | PostCard background, community card background, profile card, follower list rows |
| Input / border | `#2A2A2A` (`bg-input` / `border`) | Search bar background, text input fields, disabled button state |
| Accent (10%) | `#00E5C3` (`teal`) | Reserved elements listed below |
| Accent pressed | `#00B89A` (`teal-dark`) | Pressed state of teal-background buttons only |
| Destructive | `#FF4444` (`destructive`) | Destructive action confirmation text only — not used in Phase 5 |
| Badge | `#FF3B30` (`badge`) | Notification unread count badge background |
| Foreground | `#FFFFFF` (`foreground`) | Primary text |
| Muted | `#999999` (`muted-foreground`) | Timestamps, secondary labels, stat labels |
| Subtle | `#666666` (`subtle`) | Placeholder text, disabled icon color, search icon |

Accent (`#00E5C3`) reserved for:
1. Keyword highlight in search results (matched text background or foreground tint) — SRCH-03
2. Follow button background (when not yet following) — active follow CTA
3. Creator badge text/border in feed (existing `CreatorBadge` component pattern)
4. Community chip border on unified feed PostCard extension (community source chip)
5. Active tab indicator in community profile (Posts tab / Comments tab)

Accent is NOT used for: general link colors, list row separators, loading indicators, or passive UI chrome.

> Source: `apps/mobile/tailwind.config.js` (color tokens), `apps/mobile/components/community/CommunityCard.tsx`, `apps/mobile/components/notification/NotificationBellBadge.tsx`, `05-CONTEXT.md` § "키워드 하이라이팅"

## Screen-by-Screen Layout Contract

### Home Tab — 0-Community View

```
SafeAreaView (bg-background)
├── Header: Wecord logo (text-display, font-semibold, foreground) + NotificationBellBadge (right)
├── HomeSearchBar (px-16, height 44px, bg-input, rounded-xl)
├── PromotionBannerCarousel (width 100%, aspect 16:9, below search bar, px-0)
└── RecommendationSection
    ├── Section header: "인기 크리에이터를 만나보세요!" (px-16, text-heading font-semibold, foreground)
    └── FlatList numColumns=2 — CommunityCard items (m-8 each, bg-card rounded-xl)
```

### Home Tab — 1+-Community View (Unified Feed)

```
SafeAreaView (bg-background)
├── Header: Wecord logo + NotificationBellBadge
├── FlashList
│   ├── ListHeaderComponent:
│   │   └── PromotionBannerCarousel (width 100%, aspect 16:9)
│   └── Items: PostCard (extended with CommunityChip, px-16, mb-8)
└── Pull-to-refresh: RefreshControl (tintColor="#00E5C3")
```

### Promotion Banner Carousel

- Outer container: `width: 100%`, `aspectRatio: 16/9`
- Auto-scroll: setInterval 3000ms, clears on unmount
- Manual swipe: FlatList `horizontal pagingEnabled`
- Dot indicator: row of circles below banner, active dot `bg-teal`, inactive `bg-input`, size 6px, gap 4px
- Banner image: expo-image `contentFit="cover"`, `transition={200}`
- Tap: router.push to deeplink URL (community, external, or in-app route)
- No banners: hide carousel entirely (render null) — do not show empty area

### Community Profile Screen

```
SafeAreaView (bg-background)
├── Back button (Ionicons chevron-back, 24px, foreground, touch 44×44)
├── Avatar: 80px circle, bg-input placeholder
├── Community nickname: text-display font-semibold foreground (mt-8)
├── Stats row (mt-16, gap-24):
│   ├── [count] Posts (text-heading semibold / text-label muted — stacked)
│   ├── [count] Followers (tappable → FollowerListScreen)
│   └── [count] Following (tappable → FollowingListScreen)
├── FollowButton (mt-16) — Teal fill "팔로우" / outlined "팔로잉"
│   (Own profile: no button shown)
└── Tab bar: Posts | Comments (active: text-teal + bottom border teal 2px)
    └── FlashList: PostCard list or CommentRow list
```

### Follower / Following List Screen

```
SafeAreaView (bg-background)
├── Header: "팔로워" or "팔로잉" (text-heading semibold) + back button
└── FlatList
    └── Row (height min 60px, px-16, py-8, flex-row items-center)
        ├── Avatar: 40px circle (bg-input placeholder)
        ├── Community nickname (text-body semibold, ml-12, flex-1)
        └── FollowButton (compact, right-aligned)
```

### Post Search Screen (within community)

```
SafeAreaView (bg-background)
├── Search bar (px-16, height 44px, bg-input, rounded-xl, autofocused on mount)
└── FlashList
    └── PostCard (with HighlightedText applied to body content)
```

### Community/Creator Global Search (SRCH-01)

- Entry: tapping HomeSearchBar navigates to `/(community)/search`
- No new screen required

### In-Community Post Search Entry Point

## Interaction Contracts

### FollowButton States

| State | Visual |
|-------|--------|
| Not following | bg-teal, text-background (#000000), rounded-full, px-24 py-8 |
| Following | bg-card border border-border text-foreground rounded-full, px-24 py-8 |
| Loading | ActivityIndicator color="#00E5C3", same size/shape as button |
| Error | Revert to pre-action state; show toast or inline error text |

### Pull-to-Refresh

- Both 0-community and 1+-community home tab views support pull-to-refresh
- `RefreshControl` `tintColor="#00E5C3"` `colors={["#00E5C3"]}`

### Promotion Banner Carousel

- Auto-scroll: 3000ms interval, pauses on user swipe (onScrollBeginDrag / onScrollEndDrag)
- Swipe: FlatList `pagingEnabled horizontal`
- Dot indicator: updates on scroll, active dot width 6px teal, inactive 6px input-color
- If 1 banner: no auto-scroll, no dot indicator

### Search Debounce

- Debounce delay: 300ms (matches existing `(community)/search.tsx` pattern)
- Minimum characters to trigger search: 1 (no minimum gate — empty query shows all communities in global search; empty query shows placeholder in post search)

### Keyword Highlighting

- Match: case-insensitive substring
- Highlight: `color: "#00E5C3"` applied to matched span within `Text` component via `HighlightedText` component
- Context: applies to post body text in PostSearchScreen results only

### Skeleton Loading

- Community profile screen: show skeleton on initial load
  - Avatar: 80px circle `bg-input` (no animation — matches existing card placeholder pattern)
  - Stat row: three 32×16 `bg-input rounded` placeholders
  - Feed: 3× `bg-card rounded-xl h-24` placeholders
- Home unified feed: FlashList renders existing PostCard skeletons via existing infinite scroll loading state

### Navigation Patterns

- Avatar or nickname tap in PostCard → navigate to `/(community)/[id]/profile/[memberId]`
- Follower/Following count tap in profile → navigate to respective list screen
- Community chip tap in unified feed PostCard → navigate to `/(community)/[id]`
- Promotion banner tap → `router.push` with deeplink URL (handled by link_url field from DB)
- HomeSearchBar tap → navigate to `/(community)/search` (existing screen)
- Search icon tap in fan/artist tab → navigate to `/(community)/[id]/post-search`

## Accessibility Contract

- All Pressable elements: `accessibilityRole` declared (button / link)
- All icon-only buttons: `accessibilityLabel` in Korean with i18n key
- Touch targets: minimum 44×44px enforced on all interactive elements
- FollowButton: `accessibilityState={{ selected: isFollowing }}` to expose follow state to screen readers
- Images: `accessibilityLabel` on community cover images and promotion banners
- NotificationBellBadge: `accessibilityLabel` includes unread count (e.g. "알림, 3개 읽지 않음")
