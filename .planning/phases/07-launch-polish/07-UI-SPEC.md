# Phase 7 — UI Design Contract

> Visual and interaction contract for Phase 7 launch-polish. Generated by gsd-ui-researcher from CONTEXT.md D-01..D-37. All mobile screens reuse the established Nativewind v4 dark-theme token system introduced in Phase 01-foundation. Admin public routes (`/privacy`, `/terms`) reuse the base-nova shadcn preset locked in Phase 04-02.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Nativewind v4 (mobile) · shadcn `base-nova` (admin public routes only) |
| Preset | `tailwind.config.js` in `apps/mobile/` · `components.json` (base-nova) in `apps/admin/` |
| Component library | none on mobile (primitives from `react-native` + custom) · `@base-ui/react` on admin |
| Icon library | `@expo/vector-icons` (Ionicons, outline/filled variants) |
| Font | System default — Apple SF (iOS) · Roboto (Android) — no custom webfont shipped |

**Source of truth:**
- Existing tokens: `apps/mobile/tailwind.config.js`
- Tab layout precedent: `apps/mobile/app/(tabs)/_layout.tsx`
- CTA primitive: `apps/mobile/components/PrimaryCTAButton.tsx`

## Spacing Scale

Declared values (multiples of 4 only, matching existing `PrimaryCTAButton` + `(tabs)` conventions):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps inside tab items, radio dot inner spacing |
| sm | 8px | List row internal padding, ActionSheet option gap, notice card inner stack |
| md | 16px | Screen horizontal padding (`px-4`), list row vertical padding, section gap |
| lg | 24px | Section header bottom padding, form field vertical gap in profile edit |
| xl | 32px | Empty state vertical padding (DM coming soon icon to heading) |
| 2xl | 48px | DM Coming Soon illustration container min-height multiplier |
| 3xl | 64px | Account deletion warning top padding (alarm framing) |

**Exceptions:**
- Tab bar height: `56px` (React Native convention, already established in `_layout.tsx`)
- Primary CTA height: `52px` (locked in existing `PrimaryCTAButton`, rounded `28px` pill)
- Shop WebView control header: `48px` content + safe-area inset (D-23)
- Community/avatar circle diameter: `40dp` in More list rows (specifics §4)
- Form input height: `48px` (bio textarea uses `min-h-[96px]`, ~3 lines × 32px)

## Typography

Uses the 4-size / 2-weight system already declared in `tailwind.config.js`:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 12px | regular (400) | 1.4 | Tab bar labels, section captions, `User#XXXX` code display, field helper text |
| Body | 14px | regular (400) | 1.5 | List row titles, settings values, bio textarea input, legal page body copy |
| Heading | 16px | semibold (600) | 1.3 | Screen titles, More section headers, profile nickname, CTA button label |
| Display | 20px | semibold (600) | 1.2 | DM "Coming Soon" heading, account-deletion warning heading |

**Weight rules:**
- `regular` (400) for all body/input/helper/subtitle text
- `semibold` (600) for headings and CTAs
- No other weights allowed — medium/bold must not be introduced

**Line-height rationale:** Matches existing `fontSize` tokens. Body 1.5 keeps bio textarea legible; display 1.2 keeps marketing-style DM headline tight.

## Color

60/30/10 split matches the Phase 1 dark-theme lock:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (`bg-background`) | Screen backgrounds across More/Shop/DM tabs, legal pages, account-delete warning |
| Secondary (30%) | `#1A1A1A` (`bg-card`) | Profile card, joined-community list rows, settings rows, WebView control header, DM placeholder surface |
| Accent (10%) | `#00E5C3` (`teal`) / `#00B89A` (`teal-dark` press state) | See reserved list below |
| Destructive | `#FF4444` (`destructive`) | Log Out row label, Delete Account row label + final confirm CTA |
| Divider | `#2A2A2A` (`border`) | 1px row separators in settings / joined communities, WebView header bottom border |
| Muted text | `#999999` (`muted-foreground`) | Community nickname subtitle, section captions, "정보" labels |
| Subtle text | `#666666` (`subtle`) | Disabled states, version number, helper text |

**Accent reserved for:**
1. Active bottom-tab icon + label (already established)
2. `PrimaryCTAButton` background (DM "Notify Me", profile edit "저장", language picker "계속")
3. Selected language radio ring + inner dot (existing pattern from `language.tsx`)
4. Toggle/switch ON thumb+track (settings push-notification global switch, community per-category switches)
5. Linked/selected chevrons on list rows NOT allowed — keep chevrons `muted-foreground`
6. "Notified" success state badge in DM tab after Notify Me toggled (D-28)

**Accent NOT allowed on:**
- Tab bar background
- Body text (ever)
- Card backgrounds
- Icon strokes inside settings rows (those are `foreground` or `muted-foreground`)
- Destructive confirmations (those use `destructive`)

**Destructive reserved for:**
1. Log Out row label text
2. Delete Account row label text
3. Final "계정 삭제" confirmation CTA background (after `DELETE` typed)
4. Apple/Google sign-out error toast background

## Interaction Contracts

### 5-tab bottom navigation (D-01)

| Position | Route | Icon (Ionicons) | Label ko/en | Visible |
|----------|-------|-----------------|-------------|---------|
| 1 | `(tabs)/index` | `home-outline` / `home` | 홈 / Home | Yes |
| 2 | `(tabs)/community` | `people-outline` / `people` | 커뮤니티 / Community | Yes |
| 3 | `(tabs)/shop` | `bag-outline` / `bag` | Shop / Shop | Yes |
| 4 | `(tabs)/dm` | `chatbubbles-outline` / `chatbubbles` | DM / DM | Yes |
| 5 | `(tabs)/more` | `person-outline` / `person` | 더보기 / More | Yes |
| — | `(tabs)/notifications` | — | — | `href:null` hidden |

Active state: Teal `#00E5C3` icon + label (filled variant). Inactive: `#999999` muted-foreground (outline variant). Tab bar height `56px`, background `#000000`, top border `#2A2A2A` 1px (unchanged from existing layout).

### More tab profile card (D-05)

Row layout (height 88px, `bg-card`, rounded 12px, horizontal margin `px-4`, internal padding `p-4`):
- Left: 56px circular avatar (`expo-image`, fallback initial from nickname)
- Middle (flex): globalNickname (Heading 16px semibold) over bio preview (Body 14px regular, `muted-foreground`, single line `numberOfLines={1}`)
- Right: "프로필 편집" text button, Label 12px regular, teal — NOT an icon, per specifics §2

### Joined communities list (D-07)

- Row height 56px, 1px `border` bottom divider
- Left: 40dp circular community icon (`expo-image`)
- Middle: community name (Body 14px semibold) over `내 닉네임: {communityNickname}` (Label 12px regular `muted-foreground`)
- Right: `chevron-forward` Ionicon 16px `muted-foreground`
- Tap: `router.push(`/(community)/${id}`)`

### Settings / About / Logout sections (D-06, D-08, D-09)

Grouped list pattern (iOS settings style):
- Section header: Label 12px regular `muted-foreground`, uppercase-not-transformed, `pl-4 pt-6 pb-2`
- Row: 48px height, `bg-card`, 1px `border` divider between rows (not between sections)
- First row in group: top rounded 12px; last row: bottom rounded 12px
- Row content: left 20px Ionicons outline + Body 14px text, right value (muted) + `chevron-forward` (muted) OR `Switch` component
- Log out row: `destructive` text color, centered, no chevron
- Delete account row: `destructive` text color, in App-Info section (NOT mixed with logout), no chevron

### Profile edit screen (D-10, D-11)

- Header: native nav back arrow (left), "프로필 편집" title (center), "저장" text button (right) — disabled/subtle until dirty
- Avatar: 96dp circular, centered, tap triggers `ActionSheet` (D-11) — 4 options listed above
- Fields stacked vertically (gap 24px):
  - Nickname: text input, height 48px, `bg-card`, rounded 12px, Body 14px, character counter `muted-foreground` below right
  - Bio: multiline textarea, `min-h-[96px]`, Body 14px, auto-grow up to 6 lines, character counter `{n}/150` below right
- Keyboard avoidance: `KeyboardAvoidingView` with `behavior="padding"` iOS / `"height"` Android
- Dirty detection: compare current vs. loaded profile state; enable save only when at least one field changed

### Shop WebView (D-22..D-26)

- Header (fixed 48px content + safe-area top inset, `bg-card`, 1px `border` bottom):
  - Left: back arrow Ionicon 24px. Disabled state: `subtle` tint + `accessibilityState={{ disabled: true }}`. Enabled state: `foreground` tint.
  - Center: "Shop" label Heading 16px semibold, truncated
  - Right: refresh Ionicon 24px `foreground`
- WebView: `flex-1`, `source={{ uri: 'https://x-square.kr' }}`
- Loading: centered `ActivityIndicator` color teal while WebView emits `onLoadStart`/`onLoadEnd`
- Error fallback (D-25): centered layout — 48px `alert-circle-outline` icon `muted-foreground`, Heading 16px error heading, Body 14px error body, `PrimaryCTAButton` retry CTA
- External-link interception (D-26): `onShouldStartLoadWithRequest` returns false for non-`x-square.kr` hosts → `expo-web-browser` opens link in system browser, toast confirms

### DM Coming Soon (D-27, D-28)

- Full-screen centered layout, `flex-1 justify-center items-center`, horizontal `px-8`
- 96px `chatbubbles-outline` Ionicon, teal tint
- 24px vertical gap → Display 20px heading, centered
- 8px vertical gap → Body 14px `muted-foreground`, centered, max-width ~280px wrap
- 32px vertical gap → `PrimaryCTAButton` "Notify Me"
- After tap: optimistic UI — button swaps to "Notified" state (teal outline + teal text + checkmark icon, not filled bg). Re-tap shows toast "이미 알림이 등록되어 있어요."

### Logout confirmation (D-20)

- Uses existing `Alert.alert` imperative pattern (matches `LeaveConfirmDialog`)
- iOS: native action sheet; Android: native alert dialog; Web: `window.confirm`
- Two buttons: Cancel (default), Log out (destructive)

### Delete account flow (D-37)

Three-screen flow (all in `(more)/delete-account` group):
1. **Warning** — Display heading, body paragraph, `PrimaryCTAButton` "계속" (destructive variant: `bg-destructive`). Has cancel link back.
2. **Type-to-confirm** — Input field requiring literal `DELETE` string. `PrimaryCTAButton` "계정 삭제" disabled until match.
3. **Processing** — Fullscreen loader while `delete-user` Edge Function runs. On success: logout + replace to `/(auth)/login`. On failure: error toast + stay on screen.

### ActionSheet primitive (D-11)

- Use `@expo/react-native-action-sheet` pattern (or reuse post-more ActionSheet if already present); confirm during planning which package is installed.
- Options: 4 items + Cancel row
- Destructive index: index of "사진 삭제" (only when avatar is non-default)
- Cancel index: last option

### Language picker (D-15, D-16)

- Extract existing `(onboarding)/language.tsx` radio list into `components/settings/LanguagePicker.tsx`
- Reuse identical visual: 52px row, `bg-card`, teal border-2 when selected, radio dot right-aligned
- In settings mode: onChange immediately calls `i18n.changeLanguage()` + `supabase.from('profiles').update({ language })` — no CTA button
- In onboarding mode: preserves existing "계속" CTA

### Global push switch (D-17)

- Uses React Native `Switch` with `trackColor={{ false: '#2A2A2A', true: '#00E5C3' }}` and `thumbColor={Platform.OS === 'ios' ? '#FFFFFF' : '#FFFFFF'}`
- Toggling ON when OS permission not granted: triggers `Notifications.requestPermissionsAsync()`; if denied twice, deep-links to app settings via `Linking.openSettings()`
- State pulled from `Notifications.getPermissionsAsync()` + local profile preference

## Accessibility Baseline

- Every `Pressable` must have `accessibilityRole` and `accessibilityLabel`
- WebView back/refresh buttons: `accessibilityState={{ disabled }}` reflects WebView history
- Switches: `accessibilityRole="switch"` + value state
- ActionSheet destructive option: uses system destructive styling (red text)
- Contrast: Teal `#00E5C3` on `#000000` passes WCAG AA for Large text at Heading 16px bold. Body text MUST stay `foreground #FFFFFF` or `muted-foreground #999999` — never teal.
- Touch targets: all tap targets ≥ 44×44pt (tab bar 56px, list rows 48/56px, CTA 52px all pass)
