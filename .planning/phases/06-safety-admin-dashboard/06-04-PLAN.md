    **Data model:**
    ```typescript
    interface ReportGroup {
      target_type: 'post' | 'comment';
      target_id: string;
      report_count: number;
      reasons: string[];
      latest_report_at: string;
      status: 'pending' | 'reviewed' | 'actioned';
      // Fetched on side panel open:
      content?: string;
      author_id?: string;
      individual_reports?: Array<{ reason: string; created_at: string }>;
      sanctions?: Array<{ type: string; reason: string; created_at: string; expires_at: string | null }>;
    }
    ```

    **Report queue table (D-11):**
    - Heading: "Moderation" (20px/600)
    - Table columns per UI-SPEC: Status badge | Content preview (truncated 80 chars) | Type (post/comment) | Report count | Latest report date | Actions
    - Fetch aggregated reports: query `reports` table, group by (target_type, target_id), count reports, get latest created_at. Use supabaseAdmin:
      ```typescript
      const { data } = await supabaseAdmin
        .from('reports')
        .select('target_type, target_id, reason, status, created_at')
        .order('created_at', { ascending: false });
      ```
      Then aggregate client-side: group by (target_type, target_id), count, collect reasons, find latest date.
    - For content preview: fetch the actual post/comment content. Do a separate query for each unique target:
      ```typescript
      // For posts:
      const postIds = groups.filter(g => g.target_type === 'post').map(g => g.target_id);
      const { data: posts } = await supabaseAdmin.from('posts').select('id, content').in('id', postIds);
      // For comments:
      const commentIds = groups.filter(g => g.target_type === 'comment').map(g => g.target_id);
      const { data: comments } = await supabaseAdmin.from('comments').select('id, content').in('id', commentIds);
      ```
    - Sort: default by report count descending (most reported first)
    - Status badge: pending (yellow), reviewed (blue), actioned (green)
    - Empty state: "No pending reports" / "All reports have been reviewed. Check back later." per UI-SPEC copywriting
    - Row click: opens SidePanel with full details

    **Side panel content (D-11, D-12):**
    - Full original content text (fetched from posts/comments table including soft-deleted via service_role)
    - List of individual report reasons with counts
    - Reporter count total
    - "Take Action" section:
      - Sanction type dropdown using shadcn Select: warning / 7-day ban / 30-day ban / permanent ban (per D-12)
      - Reason textarea (required, per D-12) using shadcn Textarea
      - "Apply Sanction" button (accent color #00E5C3) — uses AlertDialog for confirmation
        - Confirmation message: "Apply {type} sanction to this user? They will be notified." per UI-SPEC
        - On confirm: insert into `user_sanctions` table with type, reason, issued_by (admin user id from supabaseBrowser.auth.getUser()), expires_at (null for warning/permanent, now+7d for 7day_ban, now+30d for 30day_ban). Then update all reports for this target to status='actioned'.
      - "Delete Content" button (destructive red #E04848) — uses AlertDialog
        - Confirmation: "Are you sure you want to delete this content? This action cannot be undone." per UI-SPEC
        - On confirm: update post/comment set deleted_at = now() (soft delete per D-14). Update reports to status='actioned', action_taken='deleted'.

    **Sanction history section (D-08, D-13):**
    - Below the "Take Action" section, show existing sanctions for the content author

    - Display as list: type badge, reason, date, expires_at
    - Below the list: "To appeal this sanction, contact support@wecord.app" text (per D-13, UI-SPEC copywriting)

    **Success feedback:**
