## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, sidebar item vertical rhythm |
| xl | 32px | Layout gaps between dashboard cards, page horizontal padding |
| 2xl | 48px | Major section breaks, sidebar top/bottom padding |
| 3xl | 64px | Page-level vertical spacing (unused in this phase) |

Exceptions:
- Sidebar width: 256px (w-64) -- fixed left sidebar per D-07
- Side panel (drawer) width: 480px -- report detail drawer per D-11
- Mobile bottom sheet: 44px minimum touch target for report reason rows

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Usage in this phase:
- **Body (14px/400):** Table cell text, report details, form inputs, sidebar menu labels, chart axis labels
- **Label (12px/400):** Badge text, table column headers, stat card labels, date range preset buttons, muted helper text. Visual distinction from body comes from the smaller size (12px vs 14px), not weight.
- **Heading (20px/600):** Page titles ("Moderation", "Analytics"), section headers, side panel title
- **Display (28px/600):** Dashboard stat card numbers (DAU, MAU), analytics highlight values

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #000000 (hsl 0 0% 0%) | Page background, main content area |
| Secondary (30%) | #1A1A1A (hsl 0 0% 10%) | Cards, sidebar background, side panel, table rows, chart tooltip background |
| Accent (10%) | #00E5C3 (hsl 170 100% 45%) | See reserved list below |
| Destructive | #E04848 (hsl 0 70% 60%) | Ban actions, content deletion, report count badges |

Accent reserved for:
- Active sidebar menu item indicator (left border or background highlight)
- Primary action buttons ("Take Action", "Save Changes", "Create")
- Chart line stroke color (primary metric line)
- Stat card trend-up indicators
- Active date range preset button

Additional semantic colors:
- **Border:** #2B2B2B (hsl 0 0% 17%) -- table borders, card borders, sidebar dividers, input borders
- **Muted foreground:** #999999 (hsl 0 0% 60%) -- secondary text, placeholder text, chart axis text
- **Warning (sanctions):** #F59E0B (amber-500) -- warning-level sanction badge only
- **Ring/Focus:** #00E5C3 -- focus ring on interactive elements (matches accent)

## Interaction Contracts

### Mobile: Report Flow

1. **Entry:** User taps `...` (more menu) on post or comment -> "Report" option appears (hidden for own content per D-05)
2. **Bottom sheet opens:** Modal slides up from bottom, semi-transparent backdrop (#000000 at 50% opacity)
3. **Reason selection:** 5 rows listed vertically (hate/spam/violence/copyright/other), each 44px tall minimum touch target
4. **"Other" flow:** Selecting "other" reveals a TextInput below the list. Submit button stays disabled until text is entered (D-03)
5. **Submit:** Tapping a reason (or submit for "other") immediately calls the report mutation
6. **Success:** Bottom sheet dismisses, toast appears: "Report has been submitted" (i18n key: `report.submitted`)
7. **Duplicate:** Bottom sheet dismisses, toast appears: "Already reported" (i18n key: `report.duplicate`) -- error code 23505
8. **Loading:** Submit button shows spinner, all reason rows disabled during mutation

### Admin: Sidebar Navigation

1. **Fixed sidebar:** 256px wide, always visible, background #1A1A1A
2. **Menu items:** Icon (20px) + label (14px/400), vertical stack with 8px gap between items, 12px vertical padding per item
3. **Active state:** Left 3px accent border (#00E5C3) + background subtle highlight (accent at 10% opacity)
4. **Hover state:** Background #2B2B2B (border color)

### Admin: Report Queue (Moderation Page)

1. **Table columns:** Status badge | Content preview (truncated 80 chars) | Type (post/comment) | Report count | Latest report date | Actions
2. **Sort:** Default by report count descending (most reported first)
3. **Row click:** Opens SidePanel (drawer) from right
4. **Side panel content:** Full original content text + media preview + list of report reasons with counts + reporter count + "Take Action" button area
5. **Take Action flow:** Dropdown select (warning / 7-day ban / 30-day ban / permanent ban) + reason textarea (required) + "Apply Sanction" button (accent) + "Delete Content" button (destructive)
6. **Confirm destructive:** AlertDialog confirmation before applying any sanction or deleting content
7. **Dismiss side panel:** "Close Panel" button (outline/ghost) or click outside the panel to dismiss without action

### Admin: Analytics Dashboard

1. **Date range:** 3 preset buttons (7d / 30d / 90d) at top-right, default 30d, active button uses accent color
2. **Stat cards row:** 4 cards across top -- DAU, WAU, MAU, New Signups (display size number + label size descriptor)
3. **Charts section:** 2 LineChart panels stacked -- (1) User activity trends (DAU line), (2) Content activity (posts/comments/reports lines, multi-line)
4. **Chart styling:** Background transparent, grid lines #333333, axis text #999999, primary line #00E5C3, secondary lines #666666 with distinct dash patterns
5. **Top communities table:** Below charts, top 10 by member count with posts/comments/reports columns
