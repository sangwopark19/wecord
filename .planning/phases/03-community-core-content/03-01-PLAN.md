-- Like count trigger on posts
CREATE OR REPLACE FUNCTION update_post_like_count() RETURNS TRIGGER AS $$
BEGIN
  IF TG_OP = 'INSERT' AND NEW.target_type = 'post' THEN
    UPDATE posts SET like_count = like_count + 1 WHERE id = NEW.target_id;
  ELSIF TG_OP = 'DELETE' AND OLD.target_type = 'post' THEN
    UPDATE posts SET like_count = GREATEST(like_count - 1, 0) WHERE id = OLD.target_id;
  END IF;
  RETURN COALESCE(NEW, OLD);
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

**Hooks:**

1. `useCommunitySearch(query: string)` — `useInfiniteQuery` with `queryKey: ['communitySearch', query]`. Uses `supabase.from('communities').select('id, slug, name, description, cover_image_url, type, category, member_count').textSearch('name', query, { type: 'websearch', config: 'simple' }).limit(20)`. Returns paginated community list. Debounce the query string input by 300ms before passing to the hook.

2. `useJoinCommunity()` — `useMutation`. Generates nickname via `supabase.functions.invoke('generate-nickname')`, then `supabase.from('community_members').insert({ user_id, community_id, community_nickname, role: 'member' })`. On 23505 error code (duplicate nickname), retry with new nickname. Invalidates `['communityMember', communityId]` on success.

3. `useCommunityMember(communityId: string)` — `useQuery` with `queryKey: ['communityMember', communityId]`. Fetches `supabase.from('community_members').select('*').eq('community_id', communityId).eq('user_id', userId).single()`. Returns null if not a member (used for gating join vs main screens).

4. `useLeaveCommunity()` — `useMutation`. Calls `supabase.from('community_members').delete().eq('id', membershipId)`. Invalidates `['communityMember', communityId]`. Navigates back to search after success.
