## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge offsets |
| sm | 8px | Avatar-to-text gaps, inline chip padding |
| md | 16px | Screen horizontal padding, card internal padding |
| lg | 24px | Section vertical spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, modal header zones |
| 3xl | 64px | Page-level spacing (e.g. scroll bottom padding for FAB clearance) |

Exceptions:
- Tab bar height: 56px (matches existing `tabBarStyle` in `_layout.tsx`)
- Touch targets (buttons, icon-only actions): minimum 44px height
- FAB button: 56px diameter (standard Material touch target)
- Post action bar icons (like, comment): 44px tap zone even if visual is 20px
- Community member avatar circles in horizontal scroll: 56px diameter
- Community card grid: 2 columns, card covers use 1:1 aspect ratio

## Typography

All values confirmed from `apps/mobile/tailwind.config.js`:

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Label | 12px | 400 regular | 1.4 | `text-label font-regular` |
| Body | 14px | 400 regular | 1.5 | `text-body font-regular` |
| Heading | 16px | 600 semibold | 1.3 | `text-heading font-semibold` |
| Display | 20px | 600 semibold | 1.2 | `text-display font-semibold` |

Notes:
- Maximum 4 sizes — do not introduce additional sizes outside this scale.
- Maximum 2 weights (400, 600) — do not use 500 or 700.
- Post body text: `text-body font-regular` (14px / 1.5)
- Community name and section titles: `text-heading font-semibold` (16px / 1.3)
- Relative timestamps, member counts, like counts: `text-label font-regular` (12px / 1.4) in `text-muted-foreground`
- Creator badge label: `text-label font-semibold` (12px / 1.4) — exception use of semibold at label size

## Color

All values confirmed from `apps/mobile/tailwind.config.js` and `apps/admin/app/globals.css`:

| Role | Tailwind token | Hex | Usage |
|------|---------------|-----|-------|
| Dominant (60%) | `bg-background` | #000000 | Screen background, tab bar background, modal backdrop behind sheet |
| Secondary (30%) | `bg-card` | #1A1A1A | Post cards, community cards, comment containers, bottom sheets, chip bar background |
| Input / border | `bg-input` / `border` | #2A2A2A | Text input backgrounds, hairline dividers between comments, card borders |
| Foreground | `text-foreground` | #FFFFFF | All primary text (nicknames, post body, headings) |
| Muted foreground | `text-muted-foreground` | #999999 | Timestamps, like counts, member counts, placeholder text |
| Subtle | `text-subtle` | #666666 | Disabled states, inactive tab labels, tertiary annotations |
| Accent (10%) | `bg-teal` / `text-teal` | #00E5C3 | Reserved-for list below |
| Accent pressed | `bg-teal-dark` | #00B89A | Teal button pressed state only |
| Destructive | `text-destructive` / `bg-destructive` | #FF4444 | Delete actions, destructive modal confirm button only |
| Notification badge | `bg-badge` | #FF3B30 | Unread notification badge dot — reserved for that single element |

**Accent (#00E5C3) is reserved for exactly these elements:**
1. Liked heart icon fill (toggled state)
2. Creator badge text (`Creator` label beside nickname in posts and comments)
3. Creator reply nickname text in comment thread
4. Active tab indicator in Fan/Artist/Highlight tab bar (underline or active text tint)
5. Active sort/filter chip selected state (background or border)
6. Primary CTA button background (`PrimaryCTAButton` component — join community, publish post)
7. Active tab in bottom navigation bar (already implemented via `tabBarActiveTintColor`)

Do not use Teal for: general links, all interactive elements generically, hover states on non-CTA elements.

## Interaction Contracts

### Like Animation
- On tap: immediately toggle icon fill and color to teal (optimistic).
- Simultaneously apply `Animated.spring` scale: `1.0 → 1.2 → 1.0`, tension 200, friction 7.
- If server returns error: revert icon state and show brief toast "좋아요 처리 실패".

### FAB Tap
- Tap FAB → navigate immediately to post composer as full-screen modal via `router.push('/(community)/compose')`.
- No intermediate action sheet.

### Post Card Image Tap
- Tap image → full-screen image viewer (lightbox), swipeable if multiple images.
- Use `expo-image` or React Native `Modal` with pan gesture; implement in Phase 3 if feasible, otherwise defer to Phase 4.

### Sort/Filter Chips
- Chip bar is horizontally scrollable, sticky below `CommunityTabBar`.
- Sort and filter are independent selections. Changing sort does not reset filter.
- Selected state: `bg-teal text-[#000000] font-semibold`. Unselected: `bg-card text-muted-foreground font-regular`.

### Pull-to-Refresh
- Standard `refreshControl` on FlashList with teal `tintColor` (`#00E5C3`).
- Triggers `refetch()` on the active feed query.

### Infinite Scroll
- Load next page when user scrolls to within 3 items of the bottom (`onEndReachedThreshold={0.3}`).
- Show skeleton loader row (3 items) at the bottom during page load.
- `FlashList estimatedItemSize={120}` for text-only posts; `getItemType` returns `'media'` vs `'text'` for optimization.

### Empty States (see Copywriting Contract below)
- Centered in the feed area; icon above text; no illustration required — Ionicons icon sufficient.

### Error States
- Network error during feed load: show error card at top of list (not full-screen). Card contains error message + "다시 시도" retry button.
- Upload failure in composer: inline error below media grid + "다시 시도" button. Do not dismiss composer.

## Accessibility

- All `Pressable` elements must include `accessibilityRole` and `accessibilityLabel` (Korean label, matching visible text or describing icon).
- `accessibilityState={{ disabled }}` on all disabled states.
- Like button: `accessibilityRole="button"`, `accessibilityLabel={liked ? "좋아요 취소" : "좋아요"}`.
- FAB: `accessibilityRole="button"`, `accessibilityLabel="새 글 쓰기"`.
- Creator badge: `accessibilityRole="text"`, `accessibilityLabel="크리에이터"`.
- Minimum touch target 44×44px enforced via `minHeight` / `minWidth` style or wrapping `Pressable` padding.
- `FlashList` items: provide `accessibilityLabel` on the top-level Pressable of each `PostCard`.
