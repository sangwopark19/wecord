**Hooks:**

1. **`useLike(targetType: 'post' | 'comment')`**:
   - `useMutation` with optimistic update
   - Parameters: `{ targetId: string, isLiked: boolean, userId: string, feedQueryKey: QueryKey }`
   - `mutationFn`: if `isLiked`, DELETE from `likes` WHERE `user_id=userId AND target_type=targetType AND target_id=targetId`. If not liked, INSERT into `likes` with `{ user_id: userId, target_type: targetType, target_id: targetId }`.
   - `onMutate`: cancel queries for `feedQueryKey`, snapshot current data, optimistically update like_count (+1 or -1) and isLiked toggle in the infinite query cache pages. For comment likes, update in the comments query cache instead.
   - `onError`: restore snapshot from context
   - `onSettled`: invalidate `feedQueryKey` to sync with server
   - Handle 23505 (duplicate like) gracefully — already liked, just ignore

2. **`useComments(postId: string)`**:
   - `useQuery` with `queryKey: ['comments', postId]`
   - Query: `supabase.from('comments').select('*, author:community_members!inner(community_nickname, id, role)').eq('post_id', postId).order('created_at', { ascending: true })`
   - After fetch, structure into thread: group by `parent_comment_id`. Root comments (parent_comment_id IS NULL) come first, then their replies (parent_comment_id = root.id) are nested below.
   - Batch-fetch isLiked for current user: `supabase.from('likes').select('target_id').eq('user_id', userId).eq('target_type', 'comment').in('target_id', commentIds)`
   - Return `{ rootComments: CommentWithReplies[] }` where `CommentWithReplies = Comment & { replies: Comment[], isLiked: boolean }`

3. **`useCreateComment()`**:
   - `useMutation`
   - Parameters: `{ postId: string, content: string, parentCommentId: string | null }`
   - Get user info from authStore: `userId`, `authorRole` from `useCommunityMember`
   - INSERT into `comments`: `{ post_id: postId, author_id: userId, parent_comment_id: parentCommentId, content, author_role: memberRole === 'creator' ? 'creator' : 'fan', is_creator_reply: memberRole === 'creator' && parentCommentId !== null }`
   - Depth guard: if the parent comment itself has a `parent_comment_id` (it's already a reply), do NOT allow — replies to replies not supported (1-depth only). Reject and show error toast.
   - On success: invalidate `['comments', postId]`
   - maxLength: 500 characters (validate before submit)

4. **`useDeleteComment()`**:
   - `useMutation`
   - Calls `supabase.from('comments').delete().eq('id', commentId)`
   - On success: invalidate `['comments', postId]`
