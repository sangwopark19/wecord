**Hooks:**

1. **`useCreatorFeed(communityId: string, artistMemberId: string | null)`**:
   - `useInfiniteQuery` with `queryKey: ['creatorFeed', communityId, artistMemberId]`
   - Query from `posts_with_nickname` (NEVER raw posts): `.eq('community_id', communityId).eq('author_role', 'creator')`
   - If `artistMemberId` is not null: add `.eq('artist_member_id', artistMemberId)` to filter to specific member
   - PAGE_SIZE = 15, cursor-based pagination identical to useFanFeed `latest` sort pattern (created_at DESC, id DESC)
   - Batch-fetch isLiked for current user (same pattern as useFanFeed)

2. **`useArtistMembers(communityId: string)`**:
   - `useQuery` with `queryKey: ['artistMembers', communityId]`
   - `supabase.from('artist_members').select('id, display_name, profile_image_url, position, sort_order, user_id').eq('community_id', communityId).order('sort_order', { ascending: true })`
   - Returns `ArtistMember[]`

3. **`useFollowMember()`**:
   - `useMutation`
   - Parameters: `{ followerCmId: string, followingCmId: string, isFollowing: boolean }`
   - If `isFollowing`: DELETE from `community_follows` where `follower_cm_id=followerCmId AND following_cm_id=followingCmId`
   - If not `isFollowing`: INSERT into `community_follows` with `{ follower_cm_id: followerCmId, following_cm_id: followingCmId }`
   - On success: invalidate `['artistMembers', communityId]` and `['communityMember', communityId]`
   - Note: MEMB-04 (push notification for followed member posts) is deferred to Phase 4 — no notification trigger here, just the follow relationship
