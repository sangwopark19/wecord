**Fix 1 — Query key mismatch in useMarkNotificationRead.ts (UAT Test 8, sub-issue 1):**

`useNotifications` uses queryKey `['notifications', user?.id, communityId]` (3 segments), but `useMarkNotificationRead` and `useMarkAllRead` use `['notifications', communityId]` (2 segments). This means cache invalidation after marking read does NOT trigger a refetch.

Update `useMarkNotificationRead` to accept both `userId` and `communityId`:
```typescript
export function useMarkNotificationRead(userId: string, communityId: string) {
```
Change ALL occurrences of `['notifications', communityId]` to `['notifications', userId, communityId]` inside both `useMarkNotificationRead` and `useMarkAllRead`. There are 6 occurrences total (3 per function: onMutate cancelQueries, onMutate getQueryData/setQueryData, onError setQueryData, onSettled invalidateQueries).

Also update `useMarkAllRead` signature to accept `(userId: string, communityId: string)`.

**Fix 2 — NotificationRow inline backgroundColor override (UAT Test 8, sub-issue 2):**

In `NotificationRow.tsx` line 58, the inline style `backgroundColor: isUnread ? 'transparent' : 'transparent'` always sets transparent, overriding the NativeWind `bg-card` / `bg-background` classes.

Remove the `backgroundColor` property entirely from the inline style object on the Pressable (line 58). Keep the rest of the inline styles (flexDirection, alignItems, minHeight, paddingHorizontal, paddingVertical). The NativeWind className `{isUnread ? 'bg-card' : 'bg-background'}` will now work correctly.
