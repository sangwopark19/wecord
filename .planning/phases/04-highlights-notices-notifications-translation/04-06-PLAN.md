In `useUnreadNotificationCount.ts`, the current query filters `.eq('community_id', communityId)` which excludes notifications where `community_id` is NULL (e.g., system notifications, or notifications created before the community_id column was backfilled).

Fix: Change both the initial query and the UPDATE re-fetch query to use an OR filter that matches either the specific community_id OR NULL community_id:

Replace:
```typescript
.eq('community_id', communityId)
```

With:
```typescript
.or(`community_id.eq.${communityId},community_id.is.null`)
```

Apply this change in TWO places:
1. The initial count query (around line 13-16)
2. The UPDATE handler re-fetch query (around line 48-51)

Root cause (UAT Test 6): Notifications created by pgmq fan-out may have NULL community_id if the trigger did not populate it, causing the badge to always show 0 in community view.
