## Spacing Scale

Declared values (multiples of 4). Applied identically to both mobile and admin surfaces.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge offset, inline padding (e.g. CreatorBadge px-1.5) |
| sm | 8px | Compact element spacing (e.g. avatar-to-text gap) |
| md | 16px | Default card padding, screen horizontal padding |
| lg | 24px | Section header padding, vertical section separation |
| xl | 32px | Major layout gaps |
| 2xl | 48px | Empty state top padding |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: minimum 44px height for all tappable interactive elements (translate button, notification row, bell icon, "See more" link). Source: iOS HIG.
- Notification badge: 20px diameter circle, offset **8px** from bell icon top-right corner (changed from 6px to align with 8-point grid).
- Compact card (Highlight section horizontal scroll): 120px wide × 160px tall — fixed size for predictable carousel layout.

Source: `apps/mobile/tailwind.config.js`, Phase 3 PostCard pattern (p-4 = 16px, gap-4 = 16px)

## Typography

### Mobile (Nativewind tokens — apps/mobile/tailwind.config.js)

| Role | Size | Weight | Line Height | Nativewind class |
|------|------|--------|-------------|-----------------|
| Label | 12px | regular (400) | 1.4 | `text-label font-regular` |
| Body | 14px | regular (400) | 1.5 | `text-body font-regular` |
| Heading | 16px | semibold (600) | 1.3 | `text-heading font-semibold` |
| Display | 20px | semibold (600) | 1.2 | `text-display font-semibold` |

Usage rules for this phase:
- **Label (12px regular)**: Relative timestamps on compact cards, notification meta text (time, community name), translation toggle link text, notice list date.
- **Body (14px regular)**: Compact card body preview (1 line, numberOfLines=1), notification message body, notice list title, translated text output.
- **Heading (16px semibold)**: Section headers in Highlight tab ("Notices", "Creator Posts", "Fan Posts", "Artist Members"), notification screen section group headers ("Today", "Yesterday", "This Week"), notice detail title.
- **Display (20px semibold)**: Screen-level titles (notification screen header, notice detail screen header).

### Admin (shadcn/Tailwind — apps/admin)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / small | 12px | regular (400) | 1.4 |
| Body | 14px | regular (400) | 1.5 |
| Heading | 16px | semibold (600) | 1.3 |
| Display | 20px | semibold (600) | 1.2 |

Admin typography follows shadcn base-nova defaults, which align with the same 4-size scale above. Admin-specific: notice form field labels use 14px regular; page titles (Create Notice, Manage Notices) use 20px semibold.

Source: `apps/mobile/tailwind.config.js` (locked), `apps/admin/app/globals.css` (shadcn CSS vars)

**Destructive (`#FF4444`) reserved exclusively for:**
1. Delete notice confirmation button (admin)
2. Error inline text (e.g. "번역에 실패했어요")

## Interaction Patterns

### Highlight Tab

- **Loading state**: Full-screen skeleton. 5 section skeletons: each shows a `HighlightSectionHeader` skeleton (gray bar 120px × 16px) + 3 CompactPostCard skeletons (120px × 160px, bg-input, rounded-xl, pulse animation via Animated.loop + Animated.sequence). Source: Claude's discretion (04-CONTEXT.md).
- **Error state**: Full-screen centered error message (body text muted-foreground) + "다시 시도" teal text button. Matches existing `community.feed.error.load` pattern.
- **Pull-to-refresh**: RefreshControl on outer ScrollView. tintColor="#00E5C3".
- **Section card tap**: CompactPostCard navigates to PostDetail screen. NoticeListCard navigates to NoticeDetail. ArtistMemberCard navigates to Artist tab filtered to that member.
- **"더보기" tap**: Creator Posts → Artist tab. Fan Posts → Fan tab. Notices → NoticeListScreen. Artist Members → Artist tab member list.

### Notifications

- **Unread badge**: Realtime channel `user:${userId}:notifications` (Supabase Realtime). Badge updates on INSERT event. Badge disappears when unread count reaches 0.
- **Loading state**: 8-row skeleton list. Each row: gray bar (full width, 56px height, bg-input, rounded-lg, pulse).
- **Row tap**: Navigate to relevant content + mark notification as read (optimistic, no confirmation).
- **Mark all read**: Single tap on header right button. Optimistic update — all rows immediately change to read visual state. Mutation on background.
- **Empty state**: Centered Ionicons notifications-outline (48px, subtle #666666) + heading + body text.
- **Notification bell entrance**: Bell icon in community header (Ionicons bell-outline, 24px, foreground, accessibilityLabel="알림"). Badge overlaid at 8px offset. Tap → push to NotificationScreen filtered to this community.

### Translation

- **Loading**: TranslateButton shows ActivityIndicator (16px teal) inline. Original text remains visible. Duration typically <1s for cached; <3s for API call.
- **Skeleton vs spinner decision**: Spinner (ActivityIndicator) — not skeleton — because the translation result length is unpredictable. Source: Claude's discretion (04-CONTEXT.md).
- **Toggle behavior**: Tapping "원문 보기" hides TranslatedTextBlock with no animation. Tapping "번역하기" again re-shows (from in-memory cache, no re-fetch). Only one translation active per post/comment at a time.
- **Cache behavior**: If `post_translations` DB record exists for this post + language, TranslateButton still shows; loading is instant (no spinner) on re-tap since result is stored in local React state after first load.

### Admin Notices

- **Schedule toggle**: Datetime input field appears/disappears with `published_immediately` boolean toggle (Switch). When schedule is set, form submit label changes from "공지 발행" to "예약 발행".
- **Pin toggle**: Switch component. Pinned notices show pin icon in list.
- **Form validation**: Client-side only — title required, community required, body required. Error messages below each field (12px destructive color).
