# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. This phase does not ship screens — it ships the **design system configuration** that all subsequent phases consume. The contract governs token values, theme configuration, and scaffolding that must be in place before any UI screen is rendered.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

Phase 1 deliverables are infrastructure:

| Deliverable | UI artifact |
|-------------|-------------|
| Nativewind v4 dark theme (`tailwind.config.js`) | Mobile token file |
| shadcn/ui init (`components.json`) in `apps/admin` | Admin design system |
| i18n skeleton (`common.json`, `auth.json` × 5 languages) | Copywriting scaffold |
| Expo SDK 55 root layout (`app/_layout.tsx`) | Theme provider wiring |

No user-facing screens are rendered in Phase 1. This contract defines the exact token values those config files must declare.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, inline badge padding |
| sm | 8px | `p-2` / `gap-2` | Chip gap, icon-to-text spacing |
| md | 16px | `p-4` / `gap-4` | Screen horizontal padding, card inner padding |
| lg | 24px | `p-6` / `gap-6` | Section vertical spacing |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between major blocks |
| 2xl | 48px | `p-12` / `gap-12` | Major section breaks |
| 3xl | 64px | `p-16` / `gap-16` | Page-level top/bottom breathing room |

Exceptions:
- Tab bar height: 56px (standard mobile safe-area-aware tab bar — not on 8pt grid but a mobile platform convention)
- FAB size: 56px × 56px (Material Design standard touch target)
- Avatar sizes: 40px (post card), 56px (community list), 80px (profile hero) — fixed by Weverse reference
- Touch target minimum: 44px × 44px on all interactive icons (Apple HIG requirement)

Source: WEVERSE-UI-UX-GUIDE §1.3 spacing values mapped to 8pt scale.

## Typography

All sizes in sp/px (Nativewind treats them equivalently on React Native).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (Regular) | 1.5 | Post body text, descriptions, list item labels |
| Label | 12px | 400 (Regular) | 1.4 | Dates, member counts, captions, tab bar labels |
| Heading | 16px | 600 (SemiBold) | 1.3 | Card titles, section headings, menu item text |
| Display | 20px | 700 (Bold) | 1.2 | Page titles, community name in header |

Weight declarations (exactly 2 weights in use per screen):
- **Regular: 400** — body text, captions, secondary info
- **SemiBold/Bold: 600–700** — headings, CTAs, nicknames, tab titles

Source: WEVERSE-UI-UX-GUIDE §1.2 typography table, collapsed to 4 roles with clean values.

Note: 28–32px display size (community hero name) is used in Phase 3+ screens. Phase 1 only configures the token — executor must declare it in `tailwind.config.js` as `fontSize.display`.

## Color

Dark theme. Fixed for all communities in MVP (per-community theme color is deferred per REQUIREMENTS.md Out-of-Scope).

| Role | Hex | Tailwind key | Usage |
|------|-----|--------------|-------|
| Dominant (60%) | `#000000` | `background` | App background, tab bar background, page background |
| Secondary (30%) | `#1A1A1A` | `card` | Cards, community list containers, modal backgrounds, bottom sheets |
| Accent (10%) | `#00E5C3` | `teal` | See reserved list below |
| Input surface | `#2A2A2A` | `input` | Text input field backgrounds, inactive filter chips, dividers |
| Text primary | `#FFFFFF` | `foreground` | All primary body text, icons (active state) |
| Text secondary | `#999999` | `muted-foreground` | Dates, member counts, secondary labels, inactive tab icons |
| Text tertiary | `#666666` | `subtle` | Description text beneath toggles, disabled states |
| Destructive | `#FF4444` | `destructive` | Logout text, destructive action text only |
| Notification badge | `#FF3B30` | `badge` | Unread notification badge dot/number |
| Divider | `#2A2A2A` | `border` | Hairline card dividers (1px) |

**Accent `#00E5C3` is reserved for:**
1. Primary CTA button background (community join button, charge button)
2. Active tab underline indicator (2px bottom border)
3. Active filter chip border and text
4. Toggle switch ON state background
5. Settings values that are actionable (language selection, content rating display)
6. Verified badge (✓) on creator accounts
7. Link text ("더 보기", hypertext)

Accent is NOT applied to: general borders, all interactive elements, hover states on non-CTA items, icons by default.

Source: ROADMAP.md success criteria ("dark theme background #000000, Teal #00E5C3 accent"), RESEARCH.md Pattern 4 tailwind.config.js, WEVERSE-UI-UX-GUIDE §1.1 color palette.

### Tailwind config declaration (mobile)

The executor MUST declare these exact values in `apps/mobile/tailwind.config.js`:

```js
colors: {
  background: '#000000',
  card: '#1A1A1A',
  input: '#2A2A2A',
  border: '#2A2A2A',
  teal: '#00E5C3',
  'teal-dark': '#00B89A',
  foreground: '#FFFFFF',
  'muted-foreground': '#999999',
  subtle: '#666666',
  destructive: '#FF4444',
  badge: '#FF3B30',
}
```

### CSS variables declaration (admin — shadcn dark mode)

The shadcn init will generate CSS variables. After init, set in `globals.css`:

```css
.dark {
  --background: 0 0% 0%;          /* #000000 */
  --card: 0 0% 10%;               /* #1A1A1A */
  --border: 0 0% 17%;             /* #2A2A2A */
  --input: 0 0% 17%;              /* #2A2A2A */
  --primary: 170 100% 45%;        /* #00E5C3 teal */
  --primary-foreground: 0 0% 0%;  /* black text on teal */
  --muted: 0 0% 17%;              /* #2A2A2A */
  --muted-foreground: 0 0% 60%;   /* #999999 */
  --destructive: 0 70% 60%;       /* #FF4444 */
  --foreground: 0 0% 100%;        /* #FFFFFF */
}
```

Phase 1 does not render user-facing screens. The copywriting contract governs what MUST be present in the i18n skeleton files created in Plan 01-03.

The executor MUST create skeleton keys in `packages/shared/i18n/locales/{lang}/common.json` and `auth.json` for these strings. Actual translated values in all 5 languages (KO/EN/TH/ZH/JA) must be present even in skeleton form (English values are acceptable as placeholders for TH/ZH/JA in Phase 1; they will be replaced in Phase 2–3).

## Interaction Patterns (Phase 1 configuration only)

These are declared in Phase 1 config and consumed by later phases.

| Pattern | Spec | Where configured |
|---------|------|-----------------|
| Tab switch animation | Cross-fade (immediate) | expo-router layout config |
| Screen push transition | Right-to-left slide | expo-router default |
| Modal / Bottom sheet | Bottom-up slide + dim overlay `rgba(0,0,0,0.5)` | react-native reusable pattern |
| Touch feedback (card) | opacity 0.7 on press | Nativewind `active:opacity-70` |
| Touch feedback (button) | scale 0.97 on press | Nativewind `active:scale-97` |
| Toggle ON color | `#00E5C3` (teal) | Tailwind `teal` token |
| Toggle OFF color | `#555555` | Tailwind custom |
