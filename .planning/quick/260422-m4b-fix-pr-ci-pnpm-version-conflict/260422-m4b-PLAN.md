# Quick Task 260422-m4b: Fix PR CI pnpm version conflict

## Context

PR #1 CI `Lint & Typecheck` job fails at the `pnpm/action-setup@v4` step with:

```
Error: Multiple versions of pnpm specified:
  - version 9 in the GitHub Action config with the key "version"
  - version pnpm@9.15.0 in the package.json with the key "packageManager"
Remove one of these versions to avoid version mismatch errors like ERR_PNPM_BAD_PM_VERSION
```

`pnpm/action-setup@v4` deprecates the `with.version` input when `packageManager` is set in `package.json` — it refuses to proceed instead of picking one. Since the downstream `Build`, `Test`, `Supabase Migration Test`, and `EAS Build` jobs all `needs: lint-typecheck`, they skip and the PR can't merge.

## Fix

Remove the `with: version: 9` block from each of the 4 `pnpm/action-setup@v4` step uses in `.github/workflows/ci.yml`. The action will read the pinned `pnpm@9.15.0` version from root `package.json` via `packageManager`.

## Tasks

### Task 1: Remove `version: 9` from all 4 pnpm/action-setup steps

- **Files**: `.github/workflows/ci.yml`
- **Action**: Delete the `with:` block (containing `version: 9`) from each of the 4 `pnpm/action-setup@v4` uses (lines 16–18, 41–43, 100–102, 123–125).
- **Verify**:
  - `grep -c "version: 9" .github/workflows/ci.yml` returns `0`

  - `yamllint` or `actionlint` (if available) still parses cleanly
