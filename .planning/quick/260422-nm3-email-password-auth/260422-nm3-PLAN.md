# Quick Task 260422-nm3: 일반 로그인 (이메일/비밀번호) 추가

## 문제 / 배경

현재 `apps/mobile/app/(auth)/login.tsx`는 Apple + Google OAuth 전용. 이메일/비밀번호 로그인 경로를 추가해야 함.

**Scope-locked:**
- 추가 only: signup + signin + forgot-password-request. OAuth 코드는 일절 건드리지 않음.
- 별도 화면 3개 (email-login, email-signup, forgot-password) — login.tsx에는 진입 버튼 1개만 추가.
- Supabase 대시보드 설정(email confirmation on/off) 변경 금지.
- DB 마이그레이션 없음 — `profiles` 테이블은 이미 UPSERT 패턴으로 email-auth 유저를 자연 처리.
- Deep-link 비밀번호 재설정 화면 없음 (follow-up).

## Context (scouted, confirmed)

- `apps/mobile/lib/supabase.ts`: PKCE + AsyncStorage storage adapter. `signInWithPassword` / `signUp` / `resetPasswordForEmail` 모두 즉시 호출 가능.
- `apps/mobile/stores/authStore.ts`:
  - `initialize()`가 `onAuthStateChange` 구독. `SIGNED_IN` 이벤트에서 `fetchOrCreateProfile(session.user.id)` 호출 → profile이 없으면 UPSERT (`onboarding_completed: false`).
  - **즉, email signup 완료 후 세션이 생기면 자동으로 profile UPSERT + onboarding 라우팅까지 AuthGuard가 처리함.** 추가 wiring 불필요.
- `apps/mobile/app/_layout.tsx` AuthGuard: `!session` → `(auth)/login`, `!onboardingCompleted` → `(onboarding)/tos`, else → `(tabs)`. 이미 동일 경로로 분기하므로 email signup 유저도 정확히 TOS로 진입.
- `packages/shared/src/i18n/locales/*/auth.json`: `login.*`, `error.auth_failed` 등 기존 키 있음. 5개 로케일 (ko/en/th/zh/ja) 전부 존재. 현재 `auth.json`에는 email UX용 키 **없음** — 이번 작업에서 추가.
- 기존 `showError()` 패턴: login.tsx L61-64 — 4초 auto-clear. 동일 패턴 재사용.
- styling: TouchableOpacity + nativewind + LinearGradient는 login.tsx의 기존 Tailwind 클래스 그대로 재사용 (bg-surface/60, border-border, rounded-[28px], h-[54px] 등).
- `fetchOrCreateProfile`(authStore.ts L56-114)은 `.single()` 사용 — 없으면 error를 반환하지만 분기 처리됨. 새 유저 UPSERT 검증됨.

2. `onSubmit` 핸들러 (signup 전용):
   ```tsx
   const [checkEmailSent, setCheckEmailSent] = useState(false);

   async function onSubmit() {
     if (!/.+@.+\..+/.test(email.trim())) {
       showError(t('error.invalid_email'));
       return;
     }
     if (password.length < 8) {
       showError(t('error.weak_password'));
       return;
     }
     setLoading(true);
     try {
       const { data, error } = await supabase.auth.signUp({
         email: email.trim(),
         password,
       });
       if (error) {
         // Supabase error messages to watch for:
         //   - 'User already registered' / 'already been registered' -> email_in_use
         //   - 'Password should be at least' -> weak_password (defensive, regex should catch)
         //   - default -> auth_failed
         const msg = error.message.toLowerCase();
         if (msg.includes('already') || msg.includes('registered')) {
           showError(t('error.email_in_use'));
         } else if (msg.includes('password')) {
           showError(t('error.weak_password'));
         } else {
           showError(t('error.auth_failed'));
         }
         return;
       }
       if (!data.session) {
         // Email confirmation is ON — user must click link in email.
         // Show 'check your email' state.
         setCheckEmailSent(true);
         return;
       }
       // Email confirmation OFF — session live.
       // onAuthStateChange -> fetchOrCreateProfile UPSERT (onboarding_completed=false)
       // -> AuthGuard routes to /(onboarding)/tos. No manual navigation.
     } catch (err) {
       showError(t('error.auth_failed'));
     } finally {
       setLoading(false);
     }
   }
   ```
